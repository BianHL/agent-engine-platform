## Обзор

Agent Engine Platform — это полностековая платформа интеллектуальных агентов, обеспечивающая полный цикл от создания агентов, управления базами знаний, оркестрации рабочих процессов и межагентного взаимодействия до аудита безопасности.

**Бэкенд**: FastAPI + Python 3.11  
**Фронтенд**: Next.js 14 + React 18 + Ant Design  
**Инфраструктура**: Оркестрация через Docker Compose

---

## Ключевые возможности

- 🤖 **Управление агентами** — Создание, настройка и публикация интеллектуальных агентов с выбором модели, системными промптами, привязкой инструментов и баз знаний
- 🔀 **Маршрутизация моделей** — Единый адаптер для OpenAI / Anthropic / Ollama и других LLM-провайдеров с балансировкой нагрузки, отказоустойчивостью и отслеживанием затрат
- 📚 **Движок знаний** — Полный RAG-конвейер: парсинг документов (PDF/Word/Excel/PPT), интеллектуальное разделение на фрагменты, векторный поиск (Milvus), полнотекстовый поиск (ES), графовый поиск (Neo4j) и двухуровневый поиск LightRAG
- ⚡ **Движок рабочих процессов** — Визуальная DAG-оркестрация с узлами LLM, условными ветвлениями, параллельным выполнением, циклами, HTTP-вызовами, песочницей кода, ручным согласованием и вложенными процессами
- 🤝 **Мультиагентное взаимодействие** — Режим Crew (последовательный / иерархический / параллельный / консенсусный) и протокол маршрутизации Handoff
- 🔧 **Движок инструментов** — Встроенные калькулятор, исполнитель кода, запросы к БД, файловые операции, HTTP-запросы, веб-поиск; поддержка регистрации пользовательских инструментов
- 🛡️ **Движок безопасности** — Проверка входных/выходных данных: защита от Prompt-инъекций, маскирование PII, фильтрация конфиденциальной информации
- 📊 **Движок оценки** — Оценка качества RAG в стиле Ragas (faithfulness/relevancy/precision/recall/tool accuracy)
- 🧠 **Система памяти** — Кратковременная память (история сессий в Redis) + долговременная память (векторное хранение + извлечение тем + реферативное сжатие)
- 🔌 **Сервис MCP** — Публикация возможностей платформы через Model Context Protocol
- 👥 **Мультитенантность** — Полная изоляция тенантов, система RBAC-прав, управление подразделениями и API-токенами
- 📝 **Аудит и мониторинг** — Журналы операций, аудит API-вызовов, отслеживание использования моделей, ограничение частоты запросов

## Быстрый старт

### Предварительные требования

- Docker & Docker Compose
- Не менее 8 ГБ свободной оперативной памяти (Milvus и Elasticsearch требуют значительных ресурсов)

### 1. Клонирование и настройка

```bash
git clone <repository-url>
cd agent-engine-platform

# Скопировать переменные окружения и изменить необходимые настройки
cp .env.example .env
# Отредактировать .env, как минимум установить:
#   DB_PASSWORD, REDIS_PASSWORD, NEO4J_PASSWORD
#   SECRET_KEY, ENCRYPTION_KEY (обязательно изменить для продакшена)
```

### 2. Запуск всех сервисов

```bash
# Полный запуск (вся инфраструктура + приложения)
docker compose --profile full up -d

# Или с внешней базой данных (только приложения + Neo4j)
docker compose --profile external-db up -d
```

### 3. Доступ

| Сервис | Адрес | Описание |
|--------|-------|----------|
| Фронтенд | http://localhost:3000 | Интерфейс управления Next.js |
| Backend API | http://localhost:8000 | Сервис FastAPI |
| Документация API | http://localhost:8000/docs | Swagger UI |
| Nginx | http://localhost:80 | Единая точка входа |
| Neo4j Browser | http://localhost:7474 | Консоль графовой БД |

### 4. Локальная разработка (без Docker)

**Бэкенд:**

```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

**Фронтенд:**

```bash
cd frontend
npm install
npm run dev
```

### Потоки данных

1. **Запрос пользователя** → Nginx → Frontend (SSR/CSR) или Backend API
2. **Запрос диалога** → Backend → Движок безопасности (проверка ввода) → Маршрутизация моделей → LLM → Движок безопасности (проверка вывода) → SSE-стриминг
3. **RAG-запрос** → Движок знаний → Парсинг документов → Разделение → Embedding → Хранение (Milvus/ES/Neo4j) → Поиск → Rerank → Генерация
4. **Асинхронные задачи** → Backend → Celery Worker (обработка документов, обучение моделей, плановая очистка)
5. **Выполнение рабочих процессов** → Движок рабочих процессов → DAG-планировщик → Выполнение узлов → Ручное согласование → Вывод результата

## Модули движков

### Движок моделей

Единый адаптационный слой LLM с поддержкой множества провайдеров:

| Адаптер | Поддерживаемые модели | Возможности |
|---------|----------------------|-------------|
| `OpenAIAdapter` | GPT-4o, GPT-4, GPT-3.5 | Chat / Streaming / Function Calling |
| `AnthropicAdapter` | Claude Sonnet / Opus / Haiku | Chat / Streaming |
| `OllamaAdapter` | Qwen2.5, Llama (локальные модели) | Chat / Streaming |

Ключевые特性: **ModelRouter** (балансировка нагрузки между провайдерами), **CircuitBreaker** (паттерн отказоустойчивости), **CostTracker** (отслеживание использования токенов и расчёт стоимости), **Мультимодальность** (адаптеры ASR/TTS/OCR/Vision)

### Движок знаний

Полный RAG-конвейер — от документа до ответа:

```
Загрузка документа → Парсинг (PDF/Word/Excel/PPT/TXT) → Интеллектуальное разделение → Embedding
    → Хранение (Milvus + ES + Neo4j) → Поиск (векторный/полнотекстовый/графовый/LightRAG) → Rerank → Генерация
```

**Режимы поиска LightRAG:**

| Режим | Описание | Применение |
|-------|----------|------------|
| `naive` | Чистое векторное сходство | Общий вопрос-ответ |
| `local` | Фокус на сущностях — извлечение конкретных имён для поиска узлов графа | Точные фактические запросы |
| `global` | Фокус на темах — извлечение широких концепций для поиска рёбер связей | Макроанализ |
| `hybrid` | Взвешенное слияние local + global через RRF | Сложные комплексные запросы |

### Движок рабочих процессов

DAG-оркестрация рабочих процессов с поддержкой следующих типов узлов:

| Тип узла | Описание |
|----------|----------|
| `llm` | Узел вызова LLM |
| `condition` | Условное ветвление |
| `parallel` | Параллельное выполнение |
| `loop` | Циклическое выполнение |
| `http` | HTTP-запрос |
| `code` | Песочница Python-кода (изоляция ресурсов) |
| `human` | Ручное согласование |
| `sub_workflow` | Вызов вложенного рабочего процесса |

Особенности: глобальный контроль тайм-аута, трассировка выполнения на уровне узлов, снимки переменных, детальные журналы выполнения.

### Мультиагентный движок

**Режим Crew**: Командная работа нескольких агентов
- Sequential (последовательный), Hierarchical (иерархический), Parallel (параллельный), Consensus (консенсусный)

**Режим Handoff**: Структурированная передача управления между агентами
- Протокол `HandoffMessage` на основе Pydantic
- `HandoffTracker` для отслеживания состояния передачи и количества переходов

### Движок инструментов

Встроенный набор инструментов:

| Инструмент | Функция |
|------------|---------|
| `calculator` | Вычисление математических выражений |
| `code_executor` | Выполнение Python-кода в песочнице |
| `db_query` | Запросы к БД (параметризованные) |
| `file_ops` | Операции чтения/записи файлов |
| `http_request` | HTTP-запросы |
| `web_search` | Веб-поиск |

Поддержка динамической регистрации пользовательских инструментов через `ToolRegistry`.

### Движок безопасности

Четырёхуровневая защита:
1. **Обнаружение Prompt-инъекций** — Регулярные выражения + семантический анализ
2. **Маскирование PII** — Автоматическое распознавание и маскирование номеров документов, телефонов, электронных почт, банковских карт
3. **Фильтрация конфиденциальной информации** — Настраиваемые уровни чувствительности (low/medium/high)
4. **Проверка соответствия** — Опциональные политики комплаенса

### Движок памяти

- **ShortTermMemory** — История сессий на основе Redis с поддержкой TTL и ограничения максимального количества сообщений
- **LongTermMemory** — Реферативное сжатие диалогов + извлечение тем + векторное хранение с поддержкой межсессионного поиска

### Движок оценки

Оценка качества RAG в стиле Ragas — 5 ключевых метрик:
- `faithfulness` — Соответствие ответа извлечённому контексту
- `answer_relevancy` — Релевантность ответа вопросу
- `context_precision` — Качество ранжирования результатов поиска
- `context_recall` — Покрытие всей необходимой информации при поиске
- `tool_call_accuracy` — Корректность вызовов инструментов

## Конфигурация

Все настройки управляются через переменные окружения, см. `.env.example`.

### Обязательные изменения для продакшена

| Параметр | Описание |

| `SAFETY_OUTPUT_CHECK_ENABLED` | true | Проверка безопасности вывода |

## Руководство по развертыванию

### Развертывание через Docker (рекомендуется)

```bash
# 1. Настройка переменных окружения
cp .env.example .env
vim .env  # Изменить все пункты с меткой <PRODUCTION>

# 2. Генерация безопасных ключей
python -c "import secrets; print(secrets.token_urlsafe(32))"

# 3. Запуск
docker compose --profile full up -d

# 4. Просмотр логов
docker compose logs -f backend
```

### Настройка HTTPS в Nginx

Отредактируйте `nginx/nginx.conf`, раскомментируйте блок HTTPS server и разместите сертификаты в директории `nginx/ssl/`:

```
nginx/ssl/cert.pem
nginx/ssl/key.pem
```

### Режим внешней базы данных

Если MySQL / Redis / Milvus / ES предоставляются внешними сервисами, настройте соответствующие URL-адреса подключения в `.env`, затем:

```bash
docker compose --profile external-db up -d
```

## Разработка и тестирование

### Тестирование бэкенда

```bash
cd backend

# Запуск всех тестов
pytest

# Только модульные тесты
pytest tests/unit -v

# Только интеграционные тесты
pytest tests/integration -v

# Миграции БД
alembic upgrade head

# Генерация миграции
alembic revision --autogenerate -m "description"
```

### Разработка фронтенда

```bash
cd frontend

npm install        # Установка зависимостей
npm run dev        # Запуск сервера разработки (:3000)
npm run build      # Продакшен-сборка
npm test           # Запуск тестов (Jest)
npm run lint       # Проверка ESLint
```

### Стандарты кодирования

| Уровень | Стандарт |
|---------|----------|
| Python | PEP 8 + Type Hints + async/await |
| TypeScript | ESLint + Strict Mode + отступ 2 пробела |
| Именование | Python: `snake_case` / TypeScript: `PascalCase` + `camelCase` |
| Коммиты | Conventional Commits (`feat(scope): description`) |
