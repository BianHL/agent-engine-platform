## 验收总则

### 验收方式

| 类别 | 验证手段 | 通过标准 |
|------|---------|---------|
| 功能验收 | 功能测试 + 集成测试 | 全部 P0/P1 用例通过 |
| 安全验收 | 安全扫描 + 渗透测试 | 无 CRITICAL/HIGH 漏洞 |
| 性能验收 | 压力测试 + 基准测试 | 满足 SLA 指标 |
| 代码验收 | 代码审查 + 静态分析 | 无 CRITICAL/HIGH 问题 |
| 文档验收 | 交付物清单核查 | 全部文档齐备 |

### 缺陷阻断标准

| 严重级别 | 定义 | 验收要求 |
|---------|------|---------|
| CRITICAL | 系统崩溃、数据丢失、安全漏洞 | 必须修复，阻断验收 |
| HIGH | 核心功能不可用、性能不达标 | 必须修复，阻断验收 |
| MEDIUM | 非核心功能异常、体验不佳 | 修复率 ≥ 80% |
| LOW | 界面瑕疵、文案问题 | 修复率 ≥ 50% |
