# 竞品功能研究报告

## Dify 功能详情

### 1. 可视化编排

| 功能 | 描述 | 亮点 |
|------|------|------|
| **Workflow 编排器** | 拖拽式 DAG 工作流设计，支持分支、循环、并行 | 低代码/无代码，非技术人员可上手 |
| **节点类型** | LLM、知识检索、代码执行、条件判断、HTTP 请求、变量聚合、模板转换 | 覆盖主流 AI 应用场景 |
| **Chatflow vs Workflow** | Chatflow 专为对话场景优化，支持多轮记忆；Workflow 用于自动化任务 | 场景化细分，降低认知负担 |
| **调试模式** | 单节点运行、逐步调试、输入输出可视化 | 开发体验优秀 |
| **版本管理** | 工作流发布版本管理，支持回滚 | 生产环境安全保障 |

### 2. 多模型支持

| 功能 | 描述 |
|------|------|
| **模型供应商** | 支持 OpenAI、Anthropic、Google、Mistral、Cohere、百度文心、阿里通义、智谱、月之暗面等 100+ 模型 |
| **模型切换** | 应用级别可随时切换底层模型，无需修改工作流 |
| **参数调节** | Temperature、Top-P、Max Tokens、Frequency Penalty 等可视化调节 |
| **模型对比** | 同一 Prompt 对比不同模型输出效果 |
| **自定义模型** | 支持接入私有部署模型（OpenAI 兼容接口） |
| **成本追踪** | Token 用量统计和成本估算 |

### 3. 知识库管理

| 功能 | 描述 |
|------|------|
| **文档支持** | TXT、Markdown、PDF、HTML、XLSX、XLS、DOCX、CSV、EML、MSG、PPTX、PPT、MDX |
| **分块策略** | 自动分块、自定义分块大小和重叠、按段落/句子分块 |
| **索引方式** | 高质量（Embedding）模式、经济（关键词）模式、混合模式 |
| **检索模式** | 向量检索、全文检索、混合检索（RRF 融合） |
| **召回测试** | 可视化测试检索效果，支持调整 Top-K、Score 阈值 |
| **多知识库** | 单应用可关联多个知识库 |
| **外部知识库** | 支持通过 API 接入外部知识库 |
| **知识图谱** | Beta 功能，支持从文档提取实体关系构建图谱 |

### 4. 应用发布

| 功能 | 描述 |
|------|------|
| **应用类型** | 聊天助手、文本生成、Agent、工作流 |
| **发布渠道** | Web App（内嵌）、API、嵌入式 iframe、WhatsApp、Slack、Discord |
| **Web App** | 自定义 Logo、欢迎语、主题色，独立访问链接 |
| **API** | RESTful API，支持流式输出，完整 SDK（Python） |
| **访问控制** | 密码保护、IP 白名单、API Key 管理 |
| **日志监控** | 对话日志、Token 用量、响应时间统计 |

### 5. 团队协作

| 功能 | 描述 |
|------|------|
| **工作空间** | 多团队隔离，独立资源配额 |
| **成员管理** | 所有者、管理员、编辑者、成员角色 |
| **应用共享** | 应用在团队内共享，支持权限控制 |
| **审核流程** | 企业版支持应用发布审核 |

### 6. API 能力

| 功能 | 描述 |
|------|------|
| **对话 API** | 创建对话、发送消息、获取历史、上传文件 |
| **补全 API** | 单次文本生成，适合非对话场景 |
| **知识库 API** | 文档 CRUD、检索 API |
| **工作流 API** | 异步执行工作流、获取执行状态 |
| **Webhook** | 对话结束回调、自定义事件通知 |
| **SDK** | Python SDK，社区贡献 JS/Go SDK |

## 行业最佳实践

### Agent 编排最佳实践

| 实践 | 说明 | 推荐方案 |
|------|------|----------|
| **模块化设计** | 将复杂任务拆分为独立模块，单一职责 | Workflow + 子工作流 |
| **渐进式复杂度** | 从简单 Prompt 开始，逐步添加工具和逻辑 | Chatflow -> Agent -> Workflow |
| **人在回路（HITL）** | 关键决策节点引入人工审核 | 条件节点 + 人工审批节点 |
| **错误处理** | 超时、重试、降级策略 | 异常节点 + 兜底回复 |
| **可观测性** | 全链路日志、Token 用量、延迟监控 | 日志系统 + 告警 |
| **Prompt 版本管理** | 系统提示词版本化，A/B 测试 | 版本管理 + 评估工具 |
| **工具调用安全** | 限制工具权限，验证参数，记录调用 | 白名单 + 参数校验 |

### 企业级部署要求

| 要求 | 说明 | 实现方式 |
|------|------|----------|
| **私有化部署** | 数据不出域，满足合规要求 | Docker/K8s 自托管 |
| **高可用** | 服务可用性 >= 99.9% | 多副本 + 负载均衡 + 健康检查 |
| **水平扩展** | 支持弹性伸缩 | 无状态设计 + 容器编排 |
| **多租户** | 团队/组织隔离 | 工作空间 + 数据隔离 |
| **审计日志** | 操作可追溯 | 全量操作日志 + 导出 |
| **SSO 集成** | 企业统一认证 | SAML/OIDC/LDAP |
| **备份恢复** | 数据定期备份，灾难恢复 | 自动备份 + 恢复流程 |

### 安全合规要求

| 要求 | 说明 | 实现方式 |
|------|------|----------|
| **数据加密** | 传输加密（TLS）、存储加密（AES-256） | HTTPS + 加密存储 |
| **访问控制** | RBAC 角色权限控制 | 角色 + 权限矩阵 |
| **PII 保护** | 敏感信息脱敏、检测、拦截 | PII 检测引擎 |
| **内容安全** | 提示词注入防护、输出内容审核 | 输入/输出双重过滤 |
| **API 安全** | 速率限制、API Key 轮换、IP 白名单 | 网关层防护 |
| **合规认证** | SOC 2、GDPR、等保三级 | 合规审计 + 认证 |
| **模型安全** | 模型调用审计、异常检测 | 调用日志 + 告警 |
| **供应链安全** | 依赖扫描、签名验证 | SCA + SBOM |

### 我们的机会点

基于以上分析，本平台的差异化机会在于：

1. **深度 Agent 能力**：Multi-Agent 协作（Crew/Hierarchical/Parallel/Consensus）+ Handoff 协议，比 Dify/Coze 更专业
2. **知识引擎深度**：知识图谱 + 向量 + 全文 + LightRAG 多模态检索，比纯向量检索更精准
3. **工作流能力**：DAG 引擎支持更复杂场景（Human Approval、Code Sandbox、Sub-workflow）
4. **安全合规**：Safety Engine（PII 检测、注入防护）内置，企业级合规
5. **私有化部署**：完整 Docker Compose 部署，数据完全自主
6. **可观测性**：内置 Eval Engine，RAG 质量可量化评估
